## Sprint 2: お客さま投げ銭フロー（UIと金額モデル / 決済はモック）

### 目的
本サービスの中心価値である「お客さまが店員さんへ感謝を送る」体験を、モック4画面に沿って動かす。Stripe の本接続は次スプリントに切り出し、ここでは画面遷移・金額計算（Model 純粋関数）・投げ銭の記録までを完成させる。決済成立はこのスプリントではダミー（モック成功）で代替し、完了画面まで通す。

### 実装する機能
- 投げ銭画面 `/tip/:staffId`（モック 01）: 顔写真・名前・店名・一言、金額3択、メッセージ（任意80字）
- 支払い方法ボトムシート（モック 03）: Apple Pay / Google Pay / カードの選択UI（sheetUp・スクリム・✕で戻る）。押下でモック決済を成立させる
- 完了画面 `/tip/:staffId/complete`（モック 04）: 「ありがとうを届けました！」、誰に・¥◯◯・メッセージ再掲、「もう一度送る」「閉じる」
- 金額計算 Model（純粋関数）: `calculateCustomerTotal` / `calculatePlatformFee` / `calculateStaffAmount`
- API: `GET /tip/:staffId`（表示情報）、`POST /tip/:staffId/intent`（tip を記録。本スプリントはモック成立で succeeded まで進める）、`GET /tip/:staffId/complete`

### スプリント契約（完了条件）
以下の全条件を満たした場合のみ、このスプリントは完了とする。

- [ ] `/tip/:staffId` を開くと店員さんの名前・店名・一言（と顔写真枠）が表示される
- [ ] 金額ボタン ¥100 / ¥300 / ¥500 が表示され、タップすると選択中ボタンがローズ塗り（背景 #ec3a6d・白文字）に変わり、他は非選択に戻る
- [ ] メッセージ入力欄に文字を入力でき、文字数カウンタが入力に応じて更新され、80文字を超えて入力できない
- [ ] 「Pay で送る」または「G Pay で送る」を押すと支払い方法ボトムシートが下からせり上がり、背面にスクリムが表示される
- [ ] ボトムシートの ✕ またはスクリムをタップするとシートが閉じ、入力（金額・メッセージ）が保持されたまま投げ銭画面に戻る
- [ ] ボトムシートの Apple Pay / Google Pay / カードのいずれかを押すと（モック決済成立を経て）完了画面へ遷移する
- [ ] 完了画面に「ありがとうを届けました！」、送り先の名前、選択した金額（¥◯◯）、入力したメッセージが再掲される
- [ ] メッセージ未入力でも投げ銭が送れ、完了画面がメッセージ枠なし（または空）で正しく表示される
- [ ] 完了画面の「もう一度送る」で投げ銭画面に戻れる
- [ ] `POST /tip/:staffId/intent` を呼ぶと tip レコードが DB に記録され、amount / platform_fee / customer_total / message / staff_id / store_id が保存される
- [ ] 金額計算 Model の純粋関数に Vitest テストがあり、`pnpm test` で全てパスする（少なくとも ¥100 / ¥300 / ¥500 の customer_total・platform_fee・staff_amount を検証）
- [ ] staff_amount（店員さんに届く満額）が選択した投げ銭額と一致し、手数料はお客さま支払額に上乗せされている（店員さんからは引かれない）ことがテストで確認できる
- [ ] 3画面すべてが最大幅 430px・中央1カラムで表示され、PC で開いても横いっぱいに広がらない
- [ ] お客さま画面はログイン・登録なしで一連のフローが完了する

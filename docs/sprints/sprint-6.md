## Sprint 6: 店の承認・スタッフ管理・感謝の可視化（金額なし）

### 目的
店員さんを守る「店承認」と、店向けの価値である「感謝の可視化」を完成させる。ここでの最重要原則は「店はお金に触れない・金額は見せない」。店向けの全経路から金額・金額ランキングを排除し、件数とお客さまの声だけを見せる。

### 実装する機能
- 店アカウント（Supabase Auth）と導入承認 `POST /store/:storeId/approve`
- スタッフ一覧管理 `GET /store/:storeId/staff`（QR発行の窓口）
- 感謝の可視化 `GET /store/:storeId/gratitude`（ありがとうの件数・お客さまの声。金額なし）
- 店ホーム（承認状態・スタッフ・感謝サマリ）

### スプリント契約（完了条件）
以下の全条件を満たした場合のみ、このスプリントは完了とする。

- [ ] 店が Supabase Auth でログインでき、店スコープの認証必須ルートが JWT を検証する
- [ ] `POST /store/:storeId/approve` で店の status が pending から approved に遷移する
- [ ] `GET /store/:storeId/staff` が所属スタッフ一覧を返す（在籍管理。QR は店員本人が発行するため店は発行主体ではない）
- [ ] `POST /store/:storeId/invites` でスタッフ招待（リンク/コード）を発行でき、店員はその招待経由で所属が確定する（方式A）
- [ ] `GET /store/:storeId/gratitude` が「ありがとうの件数」と「お客さまの声（メッセージ）」を返す
- [ ] `GET /store/:storeId/gratitude` のレスポンスに金額（amount / customer_total / platform_fee）が一切含まれない
- [ ] 店向けのどの API・画面にも金額ランキング・金額レポートが存在しない
- [ ] 感謝の可視化画面に件数とお客さまの声が表示され、金額は画面のどこにも表示されない
- [ ] 店は自店に所属するスタッフ・感謝のみ取得でき、他店のデータは取得できない
- [ ] 店向け経路に残高・着金・payout に関するデータが一切現れない（店はお金に触れない）
- [ ] 店画面が最大幅 430px・中央1カラムで表示される
- [ ] Sprint 2〜5 の機能（投げ銭フロー・店員さんの受取履歴・保留残高）が引き続き壊れず動作する

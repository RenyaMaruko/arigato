# Design Rules

UI/デザインに関わるコード変更時は、必ず以下の2つを参照すること。

1. `/docs/design-tokens.md` — カラー・フォント・スペーシング等のトンマナ（デザイントークン）
2. `/docs/design-references/` 配下の参考画面デザイン — 各画面の構成・レイアウト・デザインの完成イメージ

`/docs/design-references/` には画面ごとの参考デザイン（HTML モックや画像）を配置していく。実装する画面に対応する参考ファイルがあれば、必ず確認してから実装・修正すること。

| 画面 | 参考ファイル |
|---|---|
| ダッシュボード | `/docs/design-references/dashboard.html` |
| 村の様子 | `/docs/design-references/village.html` |
| ランキング | `/docs/design-references/ranking.html` |

> 今後、画面が増えるたびに対応する参考デザインを `/docs/design-references/` に追加していく。新しい画面を実装する際は、この一覧と `design-references/` 配下を確認すること。

## 参考デザインの再現方針

参考デザインは「完成イメージの指針」であり、ピクセル単位で完全模倣するものではない。

- **合わせるもの**: 全体の構成、レイアウト、要素の配置、デザインの方向性、トンマナ
- **合わせなくてよいもの**: ピクセル単位の余白・サイズの厳密一致
- **むしろ守るべきこと**: レスポンシブ対応など、実際の Web アプリとして使いやすい作りにする。画面幅に応じて自然に崩れず、いい感じに収まるようにする

> 参考デザインは静的な1枚絵なので固定幅で作られているが、実装では固定幅をそのまま再現せず、レスポンシブで柔軟なレイアウトにすること。「見た目の印象は同じ、でも実用的」を目指す。

## 必須ルール

- **スタイルは Tailwind CSS のユーティリティクラスで書く。インラインスタイル（`style={{...}}`）は使わない**
- カラー、フォント、スペーシング、角丸、シャドウの値は `/docs/design-tokens.md` で定義されたトークンを使う。ハードコードしない（tailwind.config に登録したトークンを使う）
- ライトUI（#eceef0 ベース）を基本とし、清潔感のあるダッシュボードデザインを維持する
- RPG 感はゴールド/クリーム（#b89a5e）のアクセントで表現する。派手なゲーム UI にしない
- 数値（レベル、EXP、ステータス等）には Outfit フォントを使い、`font-feature-settings: "tnum"` で等幅にする
- 本文・UI テキストには Noto Sans JP を使う
- カードベースのレイアウトを基本とする。カードは白背景 + 1px ボーダー + 穏やかなシャドウ
- アニメーションはクリーンで機能的に。過剰な RPG 演出禁止

## デザイントークン参照先

全てのビジュアル値は以下に定義:

| カテゴリ | 参照セクション |
|---|---|
| カラー（背景・テキスト・アクセント・ボーダー） | Color System |
| アクティビティアイコン・ステータス色 | Activity Icon Colors / Stat Colors |
| 貢献グラフ・ランキング色 | Contribution Graph Colors / Ranking Colors |
| サイドバー色 | Sidebar Colors |
| フォント（ファミリー・サイズ・ウェイト） | Typography |
| 余白 | Spacing System |
| 角丸 | Radius System |
| 影 | Shadow System |
| レイアウト・グリッド | Layout System |
| コンポーネント仕様 | Component Patterns |
| アニメーション | Motion Design |

## コンポーネントルール

### カード

- 背景: #ffffff / ボーダー: 1px solid #e9ebee / 角丸: 14px
- シャドウ: `0 1px 2px rgba(28,40,52,.05), 0 6px 20px rgba(28,40,52,.05)`
- パディング: 18px

### ボタン

- プライマリ（Cream）: 背景 #f7f1e3 / ボーダー #e3d3a8 / テキスト #876f38
- セカンダリ（Line）: 背景 #ffffff / ボーダー #e3d3a8 / テキスト #8a7338
- 角丸: 9px〜10px / フォントウェイト: 700

### テキスト階層

- メインテキスト: #2c3744 (Ink)
- サブテキスト: #4a5663 (Ink Soft)
- 補足・非アクティブ: #919ba6 (Muted)
- カードタイトル: 13px / Bold / Ink Soft / letter-spacing 0.3px

### EXP バー

- 高さ: 8px / 角丸: 6px
- グラデーション: linear-gradient(90deg, #5aa6cf, #69b98f)

## デザインの方向性

- 「Clean Dashboard x RPG Fantasy」が基本コンセプト
- エンジニアが仕事中に開いても恥ずかしくない品格
- GitHub の活動データが主役。装飾よりもデータの読みやすさを優先
- RPG の温かみはゴールドのアクセントカラーで控えめに表現

## やってはいけないこと

- デザイントークンに定義されていない色・サイズを勝手に使う
- 機能追加・ロジック変更をデザイン修正に混ぜる
- ゲーム画面のような派手な装飾（ドット絵背景、ファンタジーフォント等）
- サイドバーの配色やレイアウト構造を勝手に変更する
- Outfit フォントを数値以外に使う / Noto Sans JP を数値表示に使う
